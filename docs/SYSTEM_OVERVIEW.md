## 🔧 组件详解

### 1️⃣ LLMHarnessGenerator

**职责:** 智能生成测试harness

**工作流程:**
```
输入: CUDA kernel代码
    │
    ▼
LLM分析:
  - 函数签名
  - 参数类型
  - 线程层次
  - 算法逻辑
    │
    ▼
生成两个harness:
  ├─ CUDA harness
  └─ SYCL harness
    │
    ▼
输出: (cuda_code, sycl_code)
```

**关键特性:**
- 自动分析kernel结构
- 生成确定性输入
- 确保双平台逻辑一致
- 内置模板作为fallback

### 2️⃣ LLMErrorAnalyzer

**职责:** 智能错误诊断

**分析能力:**
```
编译错误:
  - 语法错误识别
  - API不匹配检测
  - 头文件缺失
  → 提供修复代码

运行时错误:
  - 内存访问错误
  - 线程配置错误
  - 资源分配失败
  → 提供配置建议

准确度失败:
  - 误差模式分析
  - 数值稳定性检查
  - 算法差异识别
  → 提供算法修复建议
```

**输出:** ErrorAnalysis对象
- error_type: 错误类型
- severity: 严重程度
- root_cause: 根本原因
- suggestions: 修复建议列表
- code_fixes: 具体代码修复

### 3️⃣ LLMExecutionOptimizer

**职责:** 智能优化执行策略

**优化维度:**
```
并行度优化:
  - 独立kernel并行
  - 考虑资源限制
  - 动态调整

执行顺序:
  - 高风险优先
  - 依赖关系排序
  - 失败快速反馈

分组策略:
  - 相似kernel分组
  - 资源需求分组
  - 执行时间平衡
```

**输入:** kernel列表 + 历史数据  
**输出:** ExecutionPlan

### 4️⃣ AccuracyVerifier

**职责:** 核心准确度验证

**验证流程:**
```
1. 读取输出文件
2. 数据对齐检查
3. 逐元素比较
4. 计算误差指标:
   - MAE (平均绝对误差)
   - Max Error (最大误差)
   - Pass Rate (通过率)
5. 应用容忍度判断
6. 返回验证结果
```

**容忍度策略:**
```
默认:     abs=1e-5, rel=1e-4
FP16:     abs=1e-3, rel=1e-2
Softmax:  abs=1e-3, rel=5e-3
Winograd: abs=1e-2, rel=1e-1
```
