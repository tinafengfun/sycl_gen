### Work-Group Size Findings

**WG=128 is optimal for:**
- 9/10 element-wise kernels (90%)
- All simple operations with coalesced access
- Recommended starting point for new kernels

**WG=256 is optimal for:**
- Filter/matrix transforms
- Some reduction operations
- Complex compute-intensive kernels

### Optimization Speedup

**Biggest improvements:**
- se_layer_nhwc: **+1331%** (single-thread mode)
- hard_fused_kernel: **+447%** (loop unrolling)
- add_bias_batched: **+51%** (grid-stride)

**Average speedup:** V1/V2 typically 20-50% faster than V0 baseline

## Generation Script

All new charts generated by:
```bash
python3 generate_all_charts.py
```

This script:
1. Parses all 23 result CSV files
2. Extracts peak performance metrics
3. Categorizes kernels by type
4. Compares versions (V0/V1/V2)
5. Analyzes work-group size effectiveness
6. Generates 5 comprehensive visualization charts

## Hardware Platform

All benchmarks run on:
- **GPU:** Intel Graphics [0xe211] (Battlemage G21)
- **Architecture:** Xe2 (BMG)
- **Sub-group:** 16
- **SLM:** 128 KB per work-group
- **Compiler:** icpx -fsycl -O3
